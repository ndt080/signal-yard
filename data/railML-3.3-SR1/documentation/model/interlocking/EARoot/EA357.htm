list of generic strategies for resetting a TVD section<br/>
  • Attributes
  • Advanced
  • Other Links
Attribute Scope Type
conditionalReset Public
Notes: The section will only be reset, if some conditions are fulfilled. In case of several variants one can include the condition information in the &lt;<a href="file://https://wiki3.railml.org/wiki/CO:designator">designator</a>&gt; of the reset strategy. Such conditions can be that the TVD section must be failed or the last counting action of delimiting detection points was “in”.<br/>
procedure Public
Notes: This is a placeholder for any other procedure to reset a TVD section.<br/>
sweepRunWithConfirmation Public
Notes: Here not just a sweep run is requested, but after the notification of the correct occupation and clearance sequence the operator needs to confirm the result by command before the section is reset within the interlocking.<br/>
sweepRunWithoutConfirmation Public
Notes: Especially for axle counter sections a sweep run (<i>de: Bügelfahrt</i>) can be requested. Although the TVD section reset takes place, with receipt of the operator command to the axle counter unit the interlocking keeps it logically failed until a correct occupation and clearance sequence from a passing train is notified from the axle counter. In this case the section is reset immediately without any further operator action.<br/>
unconditionalReset Public
Notes: The section will be reset on command without any conditions, i.e. after the reset command was accepted by the interlocking system the TVD section is set to clear/vacant status.<br/>
